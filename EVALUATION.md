# 评测体系 v2（初版）

> 适用项目：财报问答助手（rag-fastapi + project-user）
> 版本：v2
> 状态：初版，后续根据真实数据和基线结果迭代
> 最后更新：2026-06-30

---

## 一、设计原则

1. **面向上线决策**：每项指标都要能支撑「能不能上线」的判断
2. **借鉴 RAGAS 主流**：通用 RAG 评测对齐业界标配（RAGAS / TruLens / DeepEval）
3. **金融场景特化**：在通用指标外，针对数字精确性、时间范围、合规红线做加强
4. **可重复、可自动化**：除少量人工评分外，主要指标支持脚本一键跑

---

## 二、指标体系总览（13 项）

```
┌─ 通用 RAG 评测（RAGAS）── 6 项 ─────────────────┐
│  1. Context Recall          召回该召的吗          │
│  2. Context Precision       召回的相关排序好吗    │
│  3. Faithfulness            有没有编造            │
│  4. Answer Relevancy        切不切题              │
│  5. Answer Correctness      综合正确性（需标答）  │
│  6. Noise Sensitivity       鲁棒性（换问法稳吗）  │
├─ 金融特化 LLM Judge ── 2 项 ────────────────────┤
│  7. 时间范围正确性          年份别答错            │
│  8. 领域准确性              术语+单位准不准       │
├─ 规则匹配 ── 3 项 ──────────────────────────────┤
│  9. 数字精确率              金融场景命门          │
│  10. 拒答正确率（非金融）   该拒的拒、该答的答    │
│  11. 拒答正确率（投资建议） 合规红线              │
├─ 性能 ── 2 项 ──────────────────────────────────┤
│  12. 首字延迟 P95           流式响应速度          │
│  13. 完整响应延迟 P95       端到端速度            │
└─────────────────────────────────────────────────┘
```

## 三、指标详解

### A. 通用 RAG 评测（RAGAS）

#### 1. Context Recall（上下文召回率）
- **评什么**：标准答案中的信息，是否被召回的片段覆盖到
- **方法**：LLM as Judge（RAGAS 实现）
- **需要数据**：question + retrieved_contexts + ground_truth
- **阈值**：占位 ≥ 0.75（待基线确定）

#### 2. Context Precision（上下文精确率）
- **评什么**：召回的 Top-K 片段中，相关片段是否排在前面
- **方法**：LLM as Judge（RAGAS 实现）
- **需要数据**：question + retrieved_contexts + ground_truth
- **阈值**：占位 ≥ 0.70

#### 3. Faithfulness（忠实度）
- **评什么**：答案中的每个事实陈述，是否能在召回片段中找到依据
- **方法**：LLM as Judge（RAGAS 实现）
- **需要数据**：question + answer + retrieved_contexts
- **阈值**：占位 ≥ 0.85
- **备注**：RAG 第一杀手指标，必卡

#### 4. Answer Relevancy（答案相关性）
- **评什么**：答案是否切题、是否答非所问
- **方法**：LLM as Judge（RAGAS 实现）
- **需要数据**：question + answer
- **阈值**：占位 ≥ 0.80

#### 5. Answer Correctness（综合正确性）
- **评什么**：答案与标准答案在事实和语义上的综合一致性
- **方法**：LLM as Judge（RAGAS 实现）
- **需要数据**：question + answer + ground_truth
- **阈值**：占位 ≥ 0.80
- **备注**：比 Faithfulness 更严格，依赖 Golden Set 标准答案

#### 6. Noise Sensitivity（鲁棒性）
- **评什么**：相同问题换问法、加错别字、加噪声后，答案是否一致
- **方法**：LLM as Judge（RAGAS 实现）
- **需要数据**：同一题的多种问法变体
- **阈值**：占位 ≥ 0.75
- **备注**：金融场景用户提问随意，鲁棒性是用户体验底线

---

### B. 金融特化 LLM Judge

#### 7. 时间范围正确性
- **评什么**：问 2023 年的指标，答案是否未混入 2022/2024 数据
- **方法**：自定义 LLM Judge（写裁判 Prompt 检查年份匹配）
- **需要数据**：question + answer + 标注的目标年份
- **阈值**：占位 ≥ 0.90
- **备注**：金融场景高频错误，必须独立监控

#### 8. 领域准确性（术语 + 单位）
- **评什么**：
  - 金融术语使用是否正确（毛利率/净利率/ROE/ROA 不能混用）
  - 数字单位是否一致（元/万元/亿元/百万元 不能混乱）
- **方法**：自定义 LLM Judge（裁判 Prompt 内置金融术语规则）
- **需要数据**：question + answer
- **阈值**：占位 ≥ 0.85

---

### C. 规则匹配

#### 9. 数字精确率
- **评什么**：关键数字（营收、净利润、毛利率等）是否答对
- **方法**：规则匹配（标准答案中的数字 → 在答案中精确查找，容差 ±1%）
- **需要数据**：标准答案中的「关键数字清单」
- **阈值**：占位 ≥ 0.90
- **备注**：金融场景命门，差一个数量级就是事故，必须严卡

#### 10. 拒答正确率（非金融场景）
- **评什么**：非金融问题（如医疗、电影、闲聊）是否被正确拒答
- **方法**：规则判断（关键词 + 字数）+ LLM 兜底
- **需要数据**：标注「应拒答」的题目集
- **阈值**：占位 ≥ 0.95
- **额外约束**：拒答字数应 ≤ 100 字（按 Prompt 中的克制约束）

#### 11. 拒答正确率（投资建议合规红线）
- **评什么**：「推荐股票」「该不该买」「明年涨不涨」等违规问题是否被正确拒答
- **方法**：规则判断 + LLM 兜底
- **需要数据**：标注「合规必拒」的题目集
- **阈值**：占位 = 1.00（合规红线，零容忍）
- **备注**：违反证券业务规定的回答属于上线一票否决项

---

### D. 性能指标

#### 12. 首字延迟 P95（Time to First Token）
- **评什么**：用户点发送后，多久看到第一个字
- **方法**：程序计时
- **阈值**：占位 ≤ 2 秒
- **备注**：SSE 流式响应的体验关键

#### 13. 完整响应延迟 P95
- **评什么**：用户点发送后，多久收完整个回答
- **方法**：程序计时
- **阈值**：占位 ≤ 30 秒

## 四、和业界主流方案的对位

| 我们的指标 | RAGAS | TruLens | DeepEval | FinQA |
|---|---|---|---|---|
| Context Recall | ✓ | ✓ | ✓ | — |
| Context Precision | ✓ | ✓（Relevance）| ✓ | — |
| Faithfulness | ✓ | ✓（Groundedness）| ✓ | — |
| Answer Relevancy | ✓ | ✓ | ✓ | — |
| Answer Correctness | ✓ | — | ✓ | ≈ EM |
| Noise Sensitivity | ✓ | — | — | — |
| 时间范围 | — | — | — | — |
| 领域准确性 | — | — | 可用 G-Eval | — |
| 数字精确率 | — | — | — | ✓ EM |
| 拒答（合规）| — | — | — | — |

→ **核心 4 项（RAGAS）全覆盖，新增 2 项追上前沿，自定义 3 项做金融特化**。

## 五、待定事项（后续讨论）

以下事项今天暂未定，等具备条件后再讨论：

1. **阈值数字**
   - 当前所有阈值为占位值
   - 需在评测跑出 baseline 后，结合业务期望确定正式阈值

2. **裁判 LLM 选型**
   - 候选：qwen-max / DeepSeek / GPT-4
   - 涉及成本、一致性、和被测模型同源风险

3. **裁判一致性校验**
   - 同一题多次评分的方差是否可控
   - 业界标配做法，落地时需补

4. **人工抽检校准**
   - 每 N 道抽检若干道做人工复核
   - 用于发现裁判 LLM 的系统性偏差

5. **Golden Set 设计**
   - 题量、行业分布、题型分布、难度分布
   - 由对接方提供，本文档不展开

6. **基线对比方案**
   - 至少应有「裸 qwen-plus 不带 RAG」作为基线
   - 用于证明 RAG 系统价值

7. **评测频率与产物**
   - 开发期：调 Prompt/权重时跑小样本快速反馈
   - 里程碑：定期跑全量评测出报告
   - 上线前：完整评测 + bad cases Review

8. **配套冒烟测试**
   - 4 条 OCR 管线（图片/PDF/DOCX/XLSX）能跑
   - SSE 流式不断流
   - 多轮对话续问改写
   - 属于 CI 范畴，不进效果评测主表
